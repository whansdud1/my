# Intent - Specify

[정보화 비전·목표]
- 대학생들이 팀 프로젝트 진행 시 겪는 팀 구성의 어려움과 갈등 문제를 해결하기 위한 AI 기반 팀 매칭 플랫폼 구축
- 사용자의 성향, 협업 스타일, 역할 선호 데이터를 기반으로 최적의 팀 자동 구성 지원
- 프로젝트 수행 과정에서 역할 분담, 일정 관리, 협업 피드백 기능을 제공하여 협업 효율 향상
- 공정하고 투명한 팀 구성 환경 조성
- 팀 스트레스 감소 및 프로젝트 만족도 향상
- 장기적으로 대학 내 프로젝트 협업 표준 플랫폼으로 확장 목표

[환경·개선 동인]
- 대학 팀 프로젝트 증가에 따른 팀 구성 문제 지속 발생
- 랜덤/지인 기반 팀 구성으로 인한 역할 불균형 문제 존재
- 비대면·온라인 협업 증가로 협업 관리 플랫폼 수요 확대
- AI 기반 추천 시스템 및 데이터 분석 기술 발전
- 학생들의 협업 경험 및 성향 데이터를 활용한 맞춤형 서비스 가능
- 대학 차원의 프로젝트 운영 효율화 요구 증가

[AS-IS 기능 현황]
- 팀 프로젝트 참여 학생 정보 수집 기능
- 성별, 학년, 전공, 역할 선호 입력
- 협업 스타일 및 역량 설문 기능
- 팀 자동 매칭 기능
- 역할 균형 기반 추천
- 협업 성향 기반 추천
- 프로젝트 생성 및 팀 관리 기능
- 사용자 피드백 및 팀 평가 기능
- 협업 활동 기록 기능
- 출석, 회의 참여, 과제 수행 기록

[AS-IS 문제점·한계]
- 기존 팀 구성 방식은 주관적 판단 의존도가 높음
- 역할 편중 및 특정 학생에게 업무 집중 발생
- 협업 성향 불일치로 갈등 발생 가능성 존재
- 프로젝트 진행 과정 데이터가 체계적으로 관리되지 않음
- 사용자의 신뢰 부족 가능성
- AI 추천 기준 불투명
- 개인정보 활용 우려
- 일부 학생 소외 가능성
- 낮은 평가 점수 기반 반복 제외 위험
- 팀 활동 데이터 부족 시 추천 정확도 한계 존재

[개요]
대학생들이 팀 프로젝트 진행 시 겪는 팀 구성의 어려움과 갈등 문제를 해결하기 위한 웹 기반 팀 매칭 플랫폼
학생들의 성향, 협업 스타일, 역량 데이터를 분석하여 최적의 팀을 구성하고, 효율적인 역할 분배와 협업 환경 조성을 지원한다.


[미션]
1) 목표
- 대학생들이 팀 프로젝트 진행 시 겪는 팀 구성의 어려움과 갈등 문제를 해결
- 협업 스타일 및 업무 진행 성향을 고려한 효율적인 팀 매칭 환경을 제공
- 신뢰 기반의 공정한 협업 환경 조성에 기여

2) 방향성
- 사용자의 전공, 일정, 성향, 역할 선호 및 팀 프로젝트 참여 데이터를 분석하여 최적의 팀원을 추천
- 팀원 평가 및 피드백 시스템을 통해 원활한 의사소통과 균형 있는 역할 분배를 지원
- 향후 평가 및 피드백한 부분을 리뷰화하여 후에 다른 팀 프로젝트를 진행할 때 자기와 맞는 팀원을 구할 수 있도록 함



[핵심 활동]
1. 학생 데이터 수집 : 사용자의 기본 정보를 수집하여 팀 매칭에 활용
-사용자가 회원가입을 할 때, 성별, 학년, 전공(학과), 선호 역할(발표·기획·디자인·자료조사 등)을 기입
예시) 성별 : 남, 학년 : 3학년, 전공 : 경영학과, 선호 역할 : PPT 및 자료 조사

2. AI 기반 팀 추천 : 사용자의 역할, 일정, 성향 데이터를 기반으로 최적 팀 구성 추천
예시1) 경영학과에서 주최하는 공모전에 참여하려고 함
인원1 : 발표 및 기획 강점
인원2 : 자료 및 정보 검색을 통한 보고서 작성 능력 강점
인원3 : UI/UX 및 PPT 탬플릿 디자인 가능
인원4 :일정 관리 능력 우수
→ AI가 역할 균형을 고려해 팀을 자동으로 구성함
예시2) 공모전에 참여하려고 하나 주간에는 학과 수업으로 인하여 준비가 제한됨
→ 야간 작업 선호 사용자끼리 우선 매칭

3. 프로젝트 진행 데이터 분석 : 프로젝트 수행 과정에서 발생하는 협업 활동 데이터를 분석
-Zoom, Discord, Notion,Google workspace 등과 같은 사이트와 연동하여 협업 데이터를 분석·기록
[구체적 항목]
-회의 참여 횟수
-작업 업로드 빈도
-마감 준수 여부
-메세지 응답 회신 속도
예시) 경영학과에서 주최하는 공모전에 참여하려고 함
인원1 : 발표 및 기획 강점
인원2 : 자료 및 정보 검색을 통한 보고서 작성 능력 강점
인원3 : UI/UX 및 PPT 탬플릿 디자인 가능
인원4 :일정 관리 능력 우수
이 상황에서 인원2가 같은 팀원들에게 정보(자료)를 공유하지 않아서 공모전 준비에 차질이 생기고 있음
→ 협업 위험 신호를 표시하여 경각심 부각

4. 프로젝트 진행 데이터를 통한 리뷰 반영 : 프로젝트 종료 후 동료 평가 및 AI분석 데이터를 통한 기여도 분석 후 자기 평점에 반영
-프로젝트 종료 후 동료 평가 진행 : 기여도, 의사소통 능력, 책임감, 협업 만족도 등과 같은 항목으로 구분 후 평가 진행
평가 예시) “보고서는 잘 작성했지만, 팀원들과의 의견 공유 소통이 부족했다”, “발표를 잘 진행하였고 주도적으로 의견을 제시하여 프로젝트가 순조롭게 진행되었다”
-AI가 각 인원이 맡은 역할별 진행도, 완성도 등을 데이터로 수치화하여 평가함
→ 이러한 동료 평가 및 AI 평가를 기반으로 자기 평점에 반영하여 향후 팀원 선정에 영향

5. 리뷰를 통한 추천 시스템 개선 : 프로젝트 결과와 사용자 피드백을 기반으로 AI 추천 정확도 개선
예시1) 경영학과 공모전에 참여했던 인원들이 향후 프로젝트나 공모전 등에서 높은 만족도와 좋은 성과를 반복적으로 기록
→ AI가 해당 패턴 학습하여 이와 비슷한 프로젝트나 활동에서 우수한 성적을 기록한 인원들을 추천
예시2) 경영학과 공모전에 참여했던 인원들이 좋지 않은 성과를 지속적으로 반복하고 있음
→ AI가 해당 패턴 학습하여 추천 빈도를 줄이고 이와 비슷한 프로젝트나 활동에서 우수한 성적을 기록한 인원들을 상위에 띄워 좋은 팀 프로젝트 환경을 구축할 수 있도록 도움

6. 허위 리뷰 및 악성 평가 탐지 : 비정상적인 평가 패턴을 분석하여 악의적 리뷰 방지
예시) 경영학과 팀 프로젝트 과정에서 갈등을 빚은 특정 팀원에게만 반복적으로 낮은 점수 부여(AI 분석 데이터 확인 결과, 실제 활동 데이터는 높은데 리뷰만 매우 낮음)
→ 시스템이 이상 패턴 감지 후 관리자에게 검토 요청
→ 향후 관리자가 판단 후 해당 인원 제제

- 대학 커뮤니티 플랫폼: 학생 대상 서비스 홍보 및 사용자 유입 지원 / 학생들의 리뷰 확인을 통한 팀원 매칭 시스템
- 화상회의 및 협업 플랫폼 기업: Zoom 기반 비대면 회의 기능, 일정 공유 및 협업 시스템 연동 지원

- 대학 행정기관: 학내 시스템 연계 및 학생 대상 서비스 운영 협조

[핵심 장벽]
- 개인정보 보호 문제: 학생들의 성향, 관계, 협업 태도 등 민감한 데이터가 수집되기 때문에 정보 유출 시 사생활 침해 우려가 존재
- 데이터 신뢰성 문제: 사용자가 자신의 성향이나 협업 스타일을 실제와 다르게 입력할 경우 매칭 정확도가 낮아질 가능성이 있음.
- 알고리즘 편향 위험: 특정 성향이나 높은 평가를 받은 학생에게 팀 구성이 집중되면서 일부 학생이 반복적으로 소외될 수 있음.
- 학교 시스템 연동 문제: 대학별 LMS 및 학사 시스템 구조가 달라 플랫폼 적용 및 연동 과정에서 기술적 제약이 발생할 수 있음.
- 사용자 거부감: AI 기반 팀 매칭에 대한 불신이나 기존의 자유로운 팀 구성 방식을 선호하는 학생들의 반발 가능성이 존재

[가치제안]
- 팀 구성 시간 단축 및 효율성 향상: 학생들의 성향과 협업 데이터를 기반으로 최적의 팀을 자동 매칭하여 팀 구성 과정의 시간과 부담을 줄일 수 있음.
- 역할 불균형 감소:개인의 역량과 성향을 고려한 팀 배정을 통해 특정 인원에게 업무가 과도하게 집중되는 현상을 완화할 수 있음.
- 학생 협업 만족도 증가: 성향이 맞는 팀원 간 협업 환경을 조성하여 팀플 스트레스 감소와 만족도 향상에 기여할 수 있음.
- 교수의 관리 부담 완화: 팀 구성 및 갈등 관리 과정을 시스템이 보조함으로써 교수자의 운영 부담을 줄일 수 있음.(역할 분배 균형화, 팀원 정보 사전 공유, 활동 데이터 기반 피드백, 익명 피드백 기능)

- 협업 갈등 심화 가능성: 팀원 평가 및 피드백 과정에서 부정적인 감정이나 갈등이 발생할 가능성
- 시스템 의존성 증가: AI 기반 일정 및 역할 분배 기능에 과도하게 의존하여 자율적인 협업 능력이 감소할 가능성
