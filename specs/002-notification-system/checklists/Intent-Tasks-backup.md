# 카드 기반 디자인

콘텐츠를 독립적인 카드 형태로 구성하여 정보를 시각적으로 그룹화하는 패턴

**특징**
- 각 카드는 독립적인 정보 단위
- 반응형 그리드 배치에 적합
- 스캔하기 쉬운 구조
- 터치 친화적 인터페이스

**구현 요소**
- 일관된 카드 크기와 여백
- 그림자/테두리로 구분
- 호버/클릭 상호작용
- 이미지 + 텍스트 조합

**적용 사례**
- 상품 목록/갤러리
- 뉴스피드/블로그
- 대시보드 위젯
- 팀원/프로필 목록

**참조 사이트**
Pinterest, Trello, Dribbble, Material Design Cards
- 대표 예시: https://pinterest.com

## Core Skills

- **Frontend Design**: Vue/React 컴포넌트 설계 및 스타일링, UI 구현 전문
  - 예시: `/frontend-design "로그인 폼 컴포넌트 구현"`
- **Animation & UX**: CSS/JS 애니메이션 및 트랜지션 효과 구현
  - 예시: `/animation-ux "버튼 호버 효과 및 모달 애니메이션"`
- **Accessibility (A11y)**: 웹 접근성 표준 준수 및 ARIA 구현
  - 예시: `/accessibility "폼 접근성 개선"`
- **Database Design**: 데이터베이스 스키마 설계 및 쿼리 최적화
  - 예시: `/database-design "주문 관리 테이블 스키마"`
- **Testing**: 단위 테스트, 통합 테스트, E2E 테스트 작성
  - 예시: `/testing "로그인 기능 테스트 코드"`
- **Docker & Containers**: Docker 컨테이너화 및 docker-compose 설정
  - 예시: `/docker "Node.js 앱 Docker 컨테이너화"`
- **Network Inspector**: API 요청/응답 분석 및 네트워크 디버깅
  - 예시: `DevTools > Network 탭 활용`
- **Error Boundary**: 런타임 에러 처리 및 폴백 UI 구현
  - 예시: `ErrorBoundary 컴포넌트로 에러 격리`
- **Hot Reload Debug**: HMR(Hot Module Replacement) 문제 해결
  - 예시: `Vite HMR 로그 확인: vite.config.ts`
